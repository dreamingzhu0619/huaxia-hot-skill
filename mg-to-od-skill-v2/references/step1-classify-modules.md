# Step 1: 模块分类

## 你要做什么

给你一堆模块 JSON 文件，你要把它们分组——同一类的归到一起，不同类的分开。不需要判断哪些可变哪些固定，那是 Step 2 的事。

## 输入

1. `data/<project>/modules/_index.json` —— 模块列表和 meta 摘要
2. `data/<project>/modules/*.json` —— 每个模块的完整数据，重点看：
   - `meta.moduleName` —— 模块名称
   - `meta.position.y` —— 模块在页面中的 y 坐标（确定先后顺序）
   - `node` 树中的 **TEXT 节点文字** —— 模块里写了什么

## 分类步骤

### 第一步：看名称

直接读模块名称，理解它的语义。不要机械地去 A/B/C 后缀，而是读完之后想"这个模块在说什么"。

比如下面这些名称，读完你就知道它们全是一类——都是"产品推荐理由"：

```
产品推荐理由A
产品推荐理由B
推荐理由
产品推荐
推荐
```

还有：

```
产品卡
产品卡片
基金卡片
```

也是一类。

名称接近但业务不同怎么办？比如"热点速递"和"热点解读"，读一下它们内部的文字就能确认是不是一回事。

**这一步要回答的问题**：哪几个模块从名字上看应该是一类？

### 第二步：看内容（名称看不出来时）

如果两个模块的名称差异很大，或者名称看不出是不是同类，就去看它们内部到底写了什么。

**优先关注模块中靠上方的 TEXT**——那通常是模块的标题或核心内容，最能说明这个模块在干什么。排除掉那些 UI 标签（比如"热点速递""分享""数据来源"），它们不反映模块的业务角色。

举个例子：

模块 A 靠上方的 TEXT 是：

> "左手红利，右手低波，攻守兼备的组合底仓"

模块 B 靠上方的 TEXT 是：

> "行业龙头加持，力争稳定分红"

这两句一看就是在**推荐某个投资策略**——它们是推荐理由的标题。

再往下看，模块 A 的正文说：

> "红利策略在震荡市中提供稳定的股息收益，低波策略在市场下行时有效控制回撤……"

模块 B 的正文说：

> "红利低波指数聚焦银行等低波动板块，金融权重占比高达 51.3%……"

这两段都是在对标题做**详细解释和分析**。正文的风格、结构、在模块中的位置都一样。从语义上来说，也能说明他们两个想表达的都是为什么推荐购买这个产品，也就是推荐理由。

那结论就很清楚了：**A 和 B 是同一类模块，都是"推荐理由"。**

**这一步要回答的问题**：名称看不出是一类的模块，读内容后能不能确认它们是一类？或者两个名字看起来像一类的模块，读完内容后发现其实不是一类？

## 常见模块类型参考

这类营销长图 H5 通常有以下几种模块类型，你可以作为参照：

| 类型 | 典型名称 | 内容特征 |
|------|---------|---------|
| 营销头图 | 头图、banner、header | 品牌标题 + 副标题 + 装饰图，位于页面最顶部 |
| 热点速递 | 热点速递、热点解读、行业动态 | 资讯标题 + 摘要列表 + 数据来源 |
| 产品卡 | 产品卡、基金卡、产品卡片 | 产品名 + 代码 + 风险等级 + 收益率 + 购买按钮 |
| 推荐理由 | 推荐理由、投资策略、配置建议 | 推荐标题 + 分析正文 + 图表 + 来源 |
| 风险提示 | 风险警告、风险提示、费率说明 | 产品名列表 + 费率信息 + 合规声明 |
| 银行结束语 | 银行结束语、底部、footer | 银行 logo + slogan + 免责文字 |
| 背景 | 背景图、bg | 纯装饰，覆盖整页或大面积 |

如果当前设计稿出现了不在此列表中的模块类型，按实际内容判断即可。

## 输出

写入 `data/<project>/output/modules-classification.json`。

## 输出格式

```json
{
  "groups": [
    {
      "groupId": "recommendations",
      "groupName": "产品推荐理由",
      "role": "推荐理由",
      "roleDescription": "投资推荐分析，每篇含标题、正文、图表、数据来源",
      "moduleIndexes": [2, 3, 4],
      "moduleNames": ["产品推荐理由A", "产品推荐理由B", "产品推荐理由C"],
      "judgementBasis": "名称中都有'推荐理由'语义，内部结构均为标题→正文→图表→来源，正文内容分别解读不同投资策略"
    }
  ],
  "moduleOrder": [5, 8, 1, 2, 7, 6, 0],
  "moduleOrderNote": "按 y 坐标升序排列，背景模块（覆盖全页的）排在最后"
}
```

每个 group 加一个 `judgementBasis` 字段，简要说明你是凭什么判断它们是一类的——是看名称就看出来了，还是读了内容后确认的。
