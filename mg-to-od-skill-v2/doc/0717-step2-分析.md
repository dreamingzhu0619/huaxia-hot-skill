# Step2 内容块划分 — 分析与决策

> 日期: 2026-07-18
> 目标: 在每个 Frame（模块）中用 LAYER/PATH 的可见几何关系划分区域，识别可变 SLOT 与固定骨架

---

## 一、讨论背景

输入是一个 MasterGo 设计稿经过阶段 1（MCP 抓取 → 归一化树 → 模块拆分）之后的 `data/modules/*.json`。每个模块是一个顶层 FRAME，里面包含 LAYER（底板/色块）、PATH（装饰/图标/形状）、TEXT（文字）、嵌套的 FRAME/GROUP（容器）。

目标是把每个模块拆成：
1. **固定元素** — 品牌徽标、装饰性线条等，内容不会变的东西，直接摘掉
2. **可变区域（zone）** — 每个 zone 高度 `auto`，随填入的文字内容自动伸缩。zone 不区分为"标题区"还是"内容区"，只标记为通用 slot

---

## 二、关键发现

### 2.1 TEXT 与 LAYER 永远是平级关系

在 MasterGo 的节点树中，TEXT 不可能是 LAYER 的子节点。LAYER 是叶子节点（只有填充/描边/圆角），不能包含子元素。TEXT 和它的底板 LAYER 永远是 sibling 关系，共享同一个父 GROUP/FRAME。

### 2.2 树结构不够稳定，不能作为唯一依据

对比 CITC（中信银行）、CMB（招商银行）、huaxia-hot-industry 三个设计稿：

| 设计稿 | 模块 | 树结构特征 |
|--------|------|-----------|
| CITC | 产品推荐理由 1/2/3 | 标题在"底框"GROUP 内，正文在外面。**显式分组** |
| CMB | 推荐理由 A/B | 标题 GROUP 含 PATH+TEXT，但外层又包了一层"编组 16" |
| CMB | 市场解读 | TEXT 全部裸露在 root 层级，没有 GROUP 包裹。**隐式分组** |
| huaxia-hot-industry | 编组 54 | TEXT + LAYER + FRAME 全混在 root 下，部分有 GROUP 部分没有 |

**树是设计师操作习惯的副产品，不是稳定的结构规范。**

### 2.3 D2C HTML 的结构缺陷

D2C 是像素级绝对定位截图器，不是语义化排版引擎：
- 380 个 `position: absolute`
- 225 个 `calc()` 嵌套
- 样式属性偶发丢失（font-size 漏输出）
- 硬编码像素偏移（`top: 2.43px`）
- 无 Flex/Grid 布局

结论：D2C HTML 不能直接用来做模板还原。

### 2.4 LAYER 和 PATH 都可以作为区域骨架

LAYER 永远是矩形，但 PATH 可以是任意形状。从数据上看：

| 骨架类型 | 例子 | 来源 |
|----------|------|------|
| LAYER 矩形 | 矩形 1000（CITC 红底） | CITC 底框 |
| LAYER 矩形 | 矩形 1000 拷贝（CITC 白覆层） | CITC 底框 |
| PATH 矩形 | PATH 矩形备份 19 | huaxia-hot-industry 编组 34备份 |

**PATH 画的矩形同样可以作为区域底板**。从 bounding box 角度看，PATH 矩形和 LAYER 矩形在分区时等价。

### 2.5 children 顺序 = z-order，无显式字段

MasterGo 数据中**没有 `zIndex`、`stackOrder` 等显式层叠字段**。children 数组的顺序就是 z-order：`children[0]` 在最底层，`children[n-1]` 在最顶层。后面的覆盖前面的。遮挡关系需要从 children 顺序 + 坐标推算。

### 2.6 坐标是可靠信息源，但不能画死

`(relativeX, relativeY, width, height)` 是客观的：
- **顺序** — y 坐标确定了元素的视觉阅读顺序
- **归属** — TEXT 的 y 落在哪个 LAYER 的可见区域，就属于哪个 zone
- **间距** — 相邻元素之间的 y-gap 反映设计师的分组意图

但坐标不能画死 —— 标题长度变化会导致后续所有内容位置下移。坐标的作用是提取**顺序 + 归属 + 间距**，不是固定像素值。

### 2.7 "固定元素"陷阱

CMB 的"热点速递拷贝"GROUP 内含 LAYER + PATH + TEXT，结构上看起来像标题区（含视觉元素 + 文字）。但它实际上是品牌徽标，内容永远不会变。

| | 热点速递拷贝 | 底框（标题区） |
|---|---|---|
| 占 frame 面积 | ~5% | ~30-40% |
| 位置 | 钉在左上角 | 横跨整宽 |
| TEXT | 固定 4 字品牌名 | 可变标题内容 |
| 作用 | 品牌标识 | 承载可变内容 |

**如果不先把这类固定元素摘掉，分区算法会把它们当成 zone。**

## 三、核心规则

### 3.1 元素四分类

遍历 frame 内所有 LAYER 和 PATH，分成四类：

| 类别 | 判定条件 | 参与分区？ | 例子 |
|------|----------|-----------|------|
| **A. 蒙版/裁剪** | `mask` 不为空，或名含 `Clip`/`蒙版` | **不参与** | 矩形 1000 里的 Clip GROUP |
| **B. 纯固定元素** | 面积 < frame 8%，在边角位置，多层紧密嵌套，TEXT 极短且不变 | **不参与** | 热点速递拷贝 |
| **C. 装饰元素** | 可见面积极小（被遮挡后只剩细条/边角），无 TEXT 落入 | **不参与** | 橘色底边、金色小条、Gradient Overlay |
| **D. 骨架元素** | 有 fill，可见面积 > 阈值，有 TEXT 落在上面 | **参与** | 红底 LAYER、白覆层 LAYER、PATH 底板 |

### 3.2 可见面积计算（遮挡）

children 顺序 = z-order。从底层到顶层，逐个做矩形减法：

```
z-order:
  0: 红底     [y:10, h:110]
  1: 橘色条    [y:95, h:25]
  2: 白覆层    [y:60, h:60]

红底可见   = 红底 - 白覆层      = [y:10, h:50]
橘色条可见 = 橘色条 - 白覆层    = [y:110, h:10]  → 面积太小 → 降级为装饰
白覆层可见 = 白覆层自身         = [y:60, h:60]
```

### 3.3 分区规则

每个骨架元素的**可见矩形**就是一个 zone。合并紧邻的 zone，得到最终分区。TEXT 按 y 坐标落入对应 zone。zone 输出时不标记语义（不区分"标题区"还是"内容区"），只输出通用 slot。

### 3.4 无骨架时的 fallback

如果 frame 内没有任何骨架级 LAYER/PATH（如 CMB 市场解读）→ 用 y-gap 聚类：

```
blockGap = medianFontSize × 2.5
gap > blockGap → 新区边界
```

### 3.5 LAYER/PATH 的 CSS 转化原则

| 属性 | 用法 |
|------|------|
| `fill`（颜色/渐变/图片URL） | → CSS `background` / `background-image` |
| `borderRadius` | → CSS `border-radius` |
| `effect`（阴影/模糊） | → CSS `box-shadow` / `filter` |
| `strokeColor`, `strokeWidth` | → CSS `border` |
| `relativeX`, `relativeY` | → 转化为 CSS `padding` 或 `margin`（间距） |
| **`width`, `height`** | → **扔掉**。宽度 `100%`，高度由内部文字 `auto` 撑开 |

装饰级细小 LAYER 不建 DOM，转为 CSS 属性附着在相邻 zone 上（如橘色细条 → `border-bottom`）。

### 3.6 "拷贝"后缀 = 可重复的模板实例

`组 5803` vs `组 5803 拷贝` vs `组 5800 拷贝 2` → 同一个模板的多个实例，结构完全镜像。合并为一个列表型 zone。

### 3.7 图表区域

图表 GROUP 的特征：大量小 LAYER（柱子）+ 大量离散 TEXT（数据标签）+ "拷贝"重复。整个图表 GROUP 合并为一个 zone，不拆散。

## 四、待解决的问题

1. **可见面积阈值如何确定** — 多大算"骨架"、多小算"装饰"？需要从多个设计稿中统计 LAYER 的面积分布，找到自然分界点。

2. **PATH 非矩形形状的处理** — PATH 可以是圆形、弧线、不规则形状。目前用 bounding box 近似，是否会误判？可能需要区分"PATH 矩形"和"PATH 不规则形状"。

3. **跨层级遮挡计算** — 当前遮挡只在同层级 siblings 之间计算。跨 GROUP 边界的遮挡（如父 GROUP z-order > 叔 GROUP z-order）是否需要考虑？

4. **固定元素的自动识别** — "热点速递拷贝"目前靠面积+位置+复杂度判断。如果有其他形态的固定元素（如中间位置的固定装饰），规则是否够用？

## 五、设计决策

1. **LAYER/PATH 可见几何差集优先** — 不依赖 GROUP 层级关系（设计师可能不分组），不靠 AI 语义判断（可能误判主副标题）。用 LAYER/PATH 的可见面积做区域划分。

2. **不靠节点名字** — 设计师可能命名为"编组 12"、"矩形 1000 拷贝"等无业务含义的名字。

3. **不靠 D2C HTML** — 像素级绝对定位不可靠，缺少 font-size 等关键属性。

4. **模板输出走文档流** — LAYER 的渲染属性保留为 CSS，高度全部 `auto`，坐标数值转化为间距，不画死像素。LAYER 之间的层叠关系不 1:1 复刻 DOM 层级，而是合并为 CSS 属性。

5. **固定元素先摘除** — 品牌徽标、固定装饰（如"热点速递拷贝"）在第一关就被标记为 `fixed`，不参与后续分区。

## 六、区域划分方案：LAYER/PATH 可见几何差集法

### 6.1 核心思路

**不依赖设计师怎么建 GROUP，不靠 AI 语义判断。看设计师用 LAYER 和 PATH 画了几个"能写字的区域"。**

三个设计稿（CITC、CMB、huaxia-hot-industry）验证后发现：

| 信号 | 能告诉你什么 | 局限性 |
|------|-------------|--------|
| **LAYER/PATH 几何差集** | 大 LAYER - 小 LAYER = 子区域边界 | LAYER 不存在时就失效（如 CMB 市场解读） |
| **层级关系** | GROUP 含 LAYER+TEXT → 标题容器 | 设计师不分组时就失效 |
| **AI 语义分类** | 这段文字是标题/正文/来源 | 可能误判同层级标题（如把"左手红利"判为主标题、"攻守兼备"判为副标题） |

**优先级：LAYER/PATH 几何差集 → y-gap 聚类兜底**。AI 不作为主要判断依据。

### 6.2 总体流程

```
Module JSON (frame 节点树)
  ↓
Step 1: 元素角色分类
  把所有 LAYER + PATH 分为四类：
    A. 蒙版/裁剪 → 排除，不参与
    B. 纯固定元素 → 排除，不参与分区
    C. 装饰元素 → 排除，不参与分区
    D. 骨架元素 → 参与分区
  ↓
Step 2: 可见面积计算
  将 Step 1 筛选出的骨架 LAYER/PATH 按 children 顺序（= z-order）排列，
  从底层到顶层逐个做矩形减法，算出每个骨架元素的实际可见区域。
  ↓
Step 3: 区域划分
  每个骨架元素的可见矩形就是一个"区"。
  TEXT 按 y 坐标落入对应区域。
  ↓
Step 4: 无骨架时 fallback
  如果 frame 里没有骨架级 LAYER/PATH → y-gap 聚类。
  ↓
Step 5: 输出 Zone Map
```

### 6.3 Step 1：元素角色分类

遍历 frame 内所有 LAYER 和 PATH 节点，根据以下规则分类：

#### A. 蒙版/裁剪 → 排除

判定条件（满足任一）：
- `mask` 属性不为空
- 节点名含 `Clip`、`蒙版`、`Mask`
- 自身没有 `fill` 属性，只有子节点

处理：不参与任何后续分析。

#### B. 纯固定元素 → 排除

判定条件（需同时满足）：

| 条件 | 说明 |
|------|------|
| 可见面积占 frame < 阈值 | 面积很小（如 < 8%） |
| 位置在 frame 边角 | x 在 0~10% 或 90~100%，y 在 0~5% 或 95~100% |
| 结构高度紧凑 | 多层小 LAYER+PATH+TEXT 紧密嵌套在一个小 GROUP 内 |
| 内容固定 | 含 TEXT 极短（如 2~5 字品牌标签），且这个 TEXT 在所有同类 frame 中不变 |

典型例子："热点速递拷贝"（品牌徽标，位置始终在左上角）。

处理：直接提取样式和相对位置，标记为 `fixed`，不参与分区。

#### C. 装饰元素 → 排除

判定条件：
- 面积很小，且没有 TEXT 落在上面
- 或被遮挡后可见面积极小（如 CITC 的橘色条 矩形 1000 拷贝 2，被白覆层压住后只露出底边一条线）

典型例子：Gradient Overlay、金色小条形装饰、橘色底边细条。

处理：不参与分区，但在还原时保留为 CSS 属性（如 `border-bottom`）。

#### D. 骨架元素 → 参与分区

判定条件：
- 有 fill（颜色/渐变/图片）
- 可见面积 > 阈值
- 有 TEXT 落在其可见区域内

典型例子：矩形 1000（红底）、矩形 1000 拷贝（白覆层）、矩形备份 19（PATH 底板）。

处理：进入 Step 2。

> **注意**：PATH 和 LAYER 同等对待。PATH 画的矩形（如"编组 34备份"里的 PATH 矩形备份 19）同样可以作为骨架底板。从 bounding box 角度看，PATH 矩形和 LAYER 矩形是等价的。

### 6.4 Step 2：可见面积计算

#### 6.4.1 z-order 来源

MasterGo 数据中**没有显式的 z-index 字段**。children 数组的顺序就是 z-order：

```
children[0] = 最底层
children[1] = 第二层
...
children[n-1] = 最顶层
```

同一层级内，后面的覆盖前面的。跨层级时，子节点继承父节点的层序。

#### 6.4.2 遮挡计算

将 Step 1 筛选出的骨架 LAYER/PATH 按 z-order 从底到顶排列，逐个做矩形减法：

```
输入: 骨架元素列表 [{id, x, y, w, h}, ...] 按 z-order 升序
输出: 每个元素的可见矩形区域

for i in range(n):
    visibleRects[i] = [elements[i].rect]
    for j in range(i+1, n):  // j 在 i 上面
        visibleRects[i] = subtract(visibleRects[i], elements[j].rect)
```

示例 — CITC 底框：

```
z-order:
  0: 矩形 1000 (红底)     y:10, h:110
  1: 矩形 1000 拷贝 2 (橘) y:95, h:25   ← 设计稿画得大，但大部分被遮
  2: 矩形 1000 拷贝 (白)   y:60, h:60

计算:
  红底可见区 = 红底 - 白覆层           = [y:10, h:50]  ← 标题区
  橘色条可见区 = 橘色条 - 白覆层       = [y:110, h:10]  ← 只剩底边 → 可见面积太小，降级为装饰
  白覆层可见区 = 白覆层自身             = [y:60, h:60]  ← 副标题/过渡区
```

**阈值**：可见面积占原始面积 < 阈值（如 20%）→ 不是真正的"区"，降级为装饰元素。

### 6.5 Step 3：区域划分

每个骨架元素的**可见矩形**就是一个"候选区"。合并连续的、空隙很小的候选区，得到最终分区。

```
TEXT 按 y 坐标认领所属区域:
  for each TEXT in frame:
      for each zone in zones:
          if TEXT.y 落在 zone.visibleRect 内:
              TEXT → 属于该 zone
```

一个 zone 内的 TEXT 组成该区的可变内容 slot。

**区域的语义标签**不靠 AI 判断。输出时只标记为 `zone-0`、`zone-1` 等通用 slot，由上层业务决定填入什么内容。

### 6.6 Step 4：无骨架时的 fallback

如果 frame 内没有任何符合"骨架元素"条件的 LAYER/PATH（如 CMB 市场解读，所有元素散落在 root 且没有大面积底板）：

```
sortedElements = 所有元素按 y 排序
gaps = [elem[i+1].y - (elem[i].y + elem[i].h) for i in range(n-1)]

计算自适应阈值:
  medianFontSize = 所有 TEXT 的 fontSize 中位数
  blockGap = medianFontSize * 2.5

gap > blockGap → 新区边界
```

### 6.8 特殊情况处理

#### "拷贝"后缀 → 列表模板实例

`组 5803` / `组 5803 拷贝` / `组 5803 拷贝 2` → 同一个模板的多个实例，结构完全镜像。合并为一个列表型 zone，内部每个拷贝是一个 item slot。

#### 图表区域

图表 GROUP 的特征：大量小 LAYER（柱子）+ 大量离散 TEXT（数据标签）+ "拷贝"重复实例。整个图表 GROUP 合并为一个 `zone`，不拆散内部元素。

#### 装饰 LAYER 的还原

可见面积极小的 LAYER（如橘色底边）不单独作为 zone，而是转化为 CSS 属性附着在相邻 zone 上：
- 细条在底部 → `border-bottom`
- 细条在顶部 → `border-top`
- 角标 → `::before` / `::after` 伪元素

#### 还原时的高度联动

所有骨架 LAYER 的 `width: 100%`（或 frame 固定宽），`height: auto`，由内部文字内容自然撑开。LAYER 只提供颜色、圆角、阴影、边框等视觉属性。LAYER 之间的层叠关系用 CSS 表达，不建多层 DOM。
