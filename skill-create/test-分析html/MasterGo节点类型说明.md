# MasterGo 设计稿节点类型说明

基于 MasterGo 设计稿 API 导出的归一化节点树，所有节点类型的详细解释。

---

## 一、节点类型总览

| 类型 | 本质 | 能否装子节点 | HTML 类比 |
|------|------|-------------|-----------|
| **FRAME** | 带视觉属性的盒子容器 | 能 | `<div>` with background/border/overflow |
| **GROUP** | 透明的打包袋 | 能 | `<div style="display:contents">` 或 文件夹 |
| **LAYER** | 矩形图形元素 | 不能 | `<div>` with fill/stroke（无子元素） |
| **TEXT** | 文字 | 不能 | `<span>` 或 `<p>` |
| **PATH** | 矢量路径 | 不能 | `<path>` in SVG |
| **SVG_ELLIPSE** | 椭圆/圆 | 不能 | `<ellipse>` in SVG |

---

## 二、各节点类型详解

### 1. FRAME — 带视觉属性的盒子容器

```
[FRAME] 0:947 分享
├─ [LAYER] 0:948 矩形        ← 背景
└─ [TEXT] 0:949 分享          ← 文字
```

- **自身有视觉属性**：可以设置背景色、边框、圆角、阴影、模糊
- **自身有坐标系**：子元素定位相对于 Frame 内部坐标
- **可以裁剪溢出**：类似 `overflow: hidden`
- **支持自动布局**：类似 Flexbox（Auto Layout）
- **HTML 类比**：就是一个 `<div>`，可以设 `background-color`、`border-radius`、`overflow: hidden`

### 2. GROUP — 透明的打包袋

```
GROUP 0:56 华夏中证红利低波动ETF发起式联接C
└─ [TEXT] 0:57 华夏中证红利低波动ETF发起式联接C
```

- **自身没有任何视觉属性**：没有背景、没有边框、没有圆角、不能裁剪
- **唯一作用**：把几个元素"绑"在一起，让它们一起选中、移动、缩放、隐藏
- **来源**：设计师选中几个元素，按 Ctrl+G 编组
- **关键理解**：你在画布上看到的永远是 GROUP 里面的东西，GROUP 本身是透明的、不可见的
- **HTML 类比**：一组被包裹在 `display:contents` 容器里的元素，或理解成"文件系统的文件夹"——文件夹本身不占内容的视觉空间，只是组织单元

#### GROUP 的例子：按钮

```
GROUP 0:53 按钮                     ← 透明袋子，"按钮"只是标签名
├─ [LAYER] 0:54 矩形 18              ← 你看到的蓝色背景
└─ [TEXT] 0:55 小试一笔               ← 你看到的白色文字
```

你眼睛看到的是**蓝色矩形 + 白色文字**。GROUP 什么也没画，它只是告诉工具："选中蓝色矩形时也选文字，拖拽时一起动。"

#### GROUP 的例子：纯文字编组（设计师随手 Ctrl+G）

```
GROUP 0:56 华夏中证红利低波動ETF发起式联接C
└─ [TEXT] 0:57 华夏中证红利低波動ETF发起式联接C
```

GROUP 里只有一个 TEXT，看起来没有起到组织作用。这是设计师的操作痕迹——可能整理图层时随手编了组，或是历史编辑留下的冗余结构。GROUP 在这里仍然是一个透明的壳。

### 3. LAYER — 矩形图形元素

```
[LAYER] 0:94 矩形 1006 拷贝 14
```

- 最纯粹的矩形图形元素
- 可以有填充色、描边、圆角、阴影、模糊等视觉属性
- 是叶子节点，**不能再包含子元素**
- 名字常常是"矩形"、"矩形 23 拷贝 14"，说明是设计师画了一个矩形后不断复制
- 也可以充当**图片的载体**：当设计师在矩形上应用"图片填充"时，节点类型仍然是 LAYER（例如"位图一"、"位图二"），相当于 `background-image`
- **HTML 类比**：一个设了背景色/背景图的空 `<div>`

### 4. TEXT — 文字

```
[TEXT] 0:55 小试一笔  "小试一笔"
```

- 文本内容节点
- 属性：文字内容、字体、字号、行高、对齐、字间距、文字颜色
- 叶子节点，**不能再包含子元素**
- 文本内容支持换行（多行文本）
- **HTML 类比**：`<p>` 或 `<span>` 里面放了文字

### 5. PATH — 矢量路径

```
[PATH] 0:77 Gradient Overlay
[PATH] 0:350 路径 1
[PATH] 0:420 直线 2
```

- 用贝塞尔曲线描述的矢量图形
- 可以是一条线、一个不规则形状、一个图标
- 有填充/描边属性
- 叶子节点

#### PATH 在数据中常见的两种用途：

**a) 实际形状/线条：**
```
[PATH] 0:350 路径 1       ← 一条线或一个形状
[PATH] 0:420 直线 2       ← 一条直线
```

**b) 文字特效叠加层（Gradient Overlay / Color Overlay）：**
```
GROUP 0:75 左手红利 右手低波
├─ [TEXT] 0:76 左手红利 右手低波           ← 文字
└─ [PATH] 0:77 Gradient Overlay          ← 渐变效果
```

#### 这是什么

设计师在 MasterGo 里给文字加了"图层样式"：

```
设计师操作：
  选中文字 "左手红利 右手低波"
  → 右键 → 图层样式 → 渐变叠加
  → 选一个渐变色（比如金色 → 橙色）
```

在 MasterGo 内部，这个渐变效果是**文字的一个属性**，跟字体、字号一样属于文字本身。

#### 为什么导出后变成了独立的 PATH

MasterGo 的导出接口**没法用属性描述"渐变叠加"**——它只能表达"这个区域有渐变填充"。于是导出时把渐变从文字身上**拆出来**，变成一个独立的 PATH 节点，挂在文字旁边。

#### 为什么是 PATH 而不是 LAYER

因为渐变区域不是简单矩形——它需要**贴合文字的轮廓**（每个字的笔画形状）。PATH 可以描述任意形状，所以用它来承载这个"被裁剪到文字轮廓的渐变填充"。

本质上是导出时做了一个 **"文字轮廓的矢量蒙版 + 渐变填充"** 的组合，这个组合被存成了一个 PATH。

#### 没有加图层样式的文字对比

```
GROUP 0:64 风险等级
├─ [LAYER] 0:65 矩形 16 拷贝          ← 背景
└─ [TEXT] 0:66 较高风险                ← 没有 overlay，干净的文字
```

没有加图层样式的文字，旁边就没有多余的 PATH。只有设计师手动加了渐变/颜色叠加后，才会出现对应的 Overlay PATH。

#### 解析时如何处理

Gradient Overlay / Color Overlay 的 PATH **不是设计师画的独立图形**，是文字视觉效果的导出副产品。解析时可以把它当成**文字的颜色/渐变信息的一部分**，而不是一个独立的图形元素。

### 6. SVG_ELLIPSE — 椭圆/圆

```
[SVG_ELLIPSE] 0:261 椭圆 560
```

- 椭圆或圆形的矢量图形
- 和 PATH 类似，只是形状固定为椭圆
- 叶子节点
- **HTML 类比**：SVG `<ellipse>` 元素

### 7. Image（位图）— 没有独立的节点类型

在 MasterGo/Figma 的数据模型中，**没有独立的 IMAGE 节点类型**。

图片是作为某个矩形的"填充"存在的。就像在 CSS 里给 div 设 `background-image`，节点类型还是 div：

```
[LAYER] 0:935 位图六        ← 这是一个 LAYER（矩形），但填充是一张图片
[LAYER] 0:24 图层 738       ← 同理
```

所以在节点树里，图片表现为一个普通的 LAYER 节点，名字可能叫"位图"或"图层"。

## 三、嵌套规则

### 谁可以装子节点（容器）

- **FRAME**：可以装 FRAME、GROUP、TEXT、LAYER、PATH、SVG_ELLIPSE
- **GROUP**：可以装 FRAME、GROUP、TEXT、LAYER、PATH、SVG_ELLIPSE

### 谁不能装子节点（叶子）

- **TEXT、LAYER、PATH、SVG_ELLIPSE**：不能再装任何东西

### 嵌套关系图

```
容器节点（可以无限嵌套）
│
├── FRAME  ──→  可以包含任何类型（包括其他 FRAME）
│
└── GROUP  ──→  可以包含任何类型（包括 FRAME）

叶子节点（树的末端）
│
├── TEXT         ← 纯文字
├── LAYER        ← 矩形 / 图片填充矩形
├── PATH         ← 矢量路径 / 线条 / 特效叠加层
└── SVG_ELLIPSE  ← 椭圆 / 圆
```

## 四、FRAME vs GROUP 核心区别

| 对比维度 | FRAME | GROUP |
|---------|-------|-------|
| 自身有背景色 | 有 | 无 |
| 自身有边框 | 有 | 无 |
| 自身有圆角 | 有 | 无 |
| 可以裁剪溢出 | 可以 | 不可以 |
| 有独立坐标系 | 有 | 没有（子元素相对父级 Frame 定位） |
| 支持 Auto Layout | 支持 | 不支持 |
| 在画布上可见 | 可以（如果设了视觉属性） | 永远不可见 |
| 怎么产生的 | 设计师用 Frame 工具画的 | 设计师 Ctrl+G 编组 |

**一句话总结**：FRAME 是有视觉属性的盒子，GROUP 是纯组织用的透明塑料袋。

## 五、Frame 里面套 Frame

Frame 嵌套 Frame 跟 div 嵌套 div 是一个道理——设计师在一个 Frame 里面又画了一个 Frame。

### 例子 1：分享按钮（一层嵌套）

```
[FRAME] 0:947 分享                    ← 外层 Frame，兜住整个"分享按钮"
├─ [LAYER] 0:948 矩形                  ← 按钮背景（一个圆角矩形）
└─ [TEXT] 0:949 分享                   ← 按钮文字
```

这里设计师完全可以用一个 LAYER+TEXT 实现，但他选了用 Frame 工具。就是一个普通的容器套容器，没有任何特殊含义。

### 例子 2：买基金来招行（两层嵌套）

```
[FRAME] 0:1208 买基金 来招行              ← 整个招商银行底部卡片
├─ [LAYER] 0:1210 矩形                    ← 背景
├─ [LAYER] 0:1211 矩形                    ← 另一个背景
├─ [FRAME] 0:1213 买基金 来招行            ← 子 Frame，装 logo 图形
│  ├─ [PATH] 0:1214 形状
│  ├─ [PATH] 0:1220 形状
│  └─ ...（6 个 path）
├─ [TEXT] 0:1252 费率低 品类全 策略优 服务好
└─ [PATH] ...
```

`FRAME 0:1213` 套在 `FRAME 0:1208` 里面，是设计师把招商银行 logo 那一堆 PATH 单独装进了一个 Frame，方便整体定位管理。

### 例子 3：大产品卡（四层嵌套）

```
[FRAME] 0:1368 产品卡                      ← 卡片外层
└─ [FRAME] 0:1370 大产品卡                  ← 卡片内层
   ├─ [FRAME] 0:1371 按钮2备份              ← 按钮
   │  ├─ [FRAME] 0:1372 矩形 + 矩形蒙版      ← 按钮背景装饰
   │  │  ├─ [PATH] 0:1373 蒙版
   │  │  └─ ...
   │  ├─ [TEXT] 0:1379 小试一笔
   │  └─ [FRAME] 0:1380 手                  ← 按钮上的手型图标
   │     ├─ [SVG_ELLIPSE] ...
   │     └─ [FRAME] 0:1384 手               ← 图标内部的子 Frame
   │        ├─ [PATH] 0:1385 蒙版
   │        └─ ...
   ├─ [TEXT] 0:1425 华夏中证电网...
   └─ [FRAME] 0:1427 标签                   ← 基金代码+风险等级
```

### 本质

Frame 套 Frame 跟 Group 套 Group 在组织思路上没有区别——都是设计师在拆分层级结构。唯一不同的是 Frame 本身能带背景、边框、圆角等视觉属性，而 Group 是透明的。

用 HTML 来表达，Frame 嵌套就是：

```html
<div class="card">           <!-- FRAME 产品卡 -->
  <div class="card-inner">   <!-- FRAME 大产品卡 -->
    <div class="button">     <!-- FRAME 按钮 -->
      <span>小试一笔</span>   <!-- TEXT -->
    </div>
  </div>
</div>
```

每个 `<div>` 都可以有自己的样式——Frame 套 Frame 就是这么个东西。

## 六、不同设计师的结构差异

同一套工具（MasterGo），同一个导出接口，不同设计师产出的节点树结构可能完全不同：

### 设计师 A（用 GROUP 组织）

```
[FRAME] 产品卡
└─ [GROUP] 产品卡              ← 喜欢用 GROUP
   ├─ [LAYER] 矩形...
   ├─ [GROUP] 按钮
   │  ├─ [LAYER] 矩形
   │  └─ [TEXT] 小试一笔
   └─ [TEXT] 近一年涨跌幅       ← TEXT 在 GROUP 下
```

### 设计师 B（用 FRAME 嵌套）

```
[FRAME] 头图banner
├─ [FRAME] 编组 7              ← 喜欢用 FRAME
├─ [TEXT] 投资落地+出口激增     ← TEXT 直接在 FRAME 下
├─ [FRAME] 分享               ← FRAME 里套 FRAME
│  ├─ [LAYER] 矩形
│  └─ [TEXT] 分享
└─ [LAYER] 位图六
```

**解析注意**：不要假定"TEXT 一定在 GROUP 里"或"FRAME 下面一定有 GROUP"，结构完全取决于设计师的操作习惯。
