# 组件与 Schema 清单（待确认草案）

> 目的：把「一模块一份绝对定位 CSS」重构为「一种组件类型一份自适应 CSS + 一个 JSON schema」。
> 每期换内容 = 填 JSON；同一种组件可复制 N 份，高度随内容自适应。

## 判断「是不是同一种组件」的方法（三信号合一）

不能只看骨架，也不能只看名字。三条一起看：

1. **骨架签名**：背景（渐变/底色）、圆角/边框/阴影、标题框、字体色号体系、内部布局。
2. **frame 名字词根**：如「推荐理由A」「推荐理由B」共享词根「推荐理由」。
3. **正文语义 + 标题装饰**：正文在讲同一类事（都在讲"产品为什么好"），且标题附近的小装饰组件一致。

三者一致 → 判为同一种组件，合并成一份 CSS + 一个 schema，实例靠 JSON 区分。

## 组件清单（7 种）

标注 `singleton` = 每页一个；`repeatable` = 每期可复制 N 份、高度自适应。

### 1. `banner` — 头图（singleton）
固定：背景图、「热点速递」角标、装饰光点、分享按钮。
```json
{ "titleLine1": "投资落地+出口激增", "titleLine2": "电网进入高景气周期" }
```

### 2. `hotspot-frontline` — 热点前线（singleton）
卡骨架（蓝渐变白卡）+ 标题「热点前线」(带装饰) + 富文本正文 + 来源行。
```json
{ "title": "热点前线", "body": "截止3月5日收盘…", "source": "信息来源：东方财富资讯，日期截止2026.3.5" }
```

### 3. `market-analysis` — 市场解读（singleton）
卡骨架 + 标题 + 导语 + 若干分论点 + 免责标注。
小标题（从国内景气度看/从政策环境看/从海外需求看）是**固定结构**，正文可变。
```json
{
  "title": "市场解读",
  "intro": "随着"十五五"规划落地…",
  "points": [
    { "heading": "从政策环境看", "body": "行业处于政策高度友好…" },
    { "heading": "从海外市场需求上看", "body": "近期美国迎来史诗级…" }
  ],
  "disclaimer": "以上观点由华夏基金提供"
}
```

### 4. `recommendation` — 推荐理由（repeatable 1..N）★核心复用组件
= 原推荐理由 A + B 合并。卡骨架 + 标题(带`编组8`装饰) + 正文 + 图表 + 资料来源。
**每期几个推荐理由就复制几份**，各自填内容，高度随正文/图表自适应。
```json
{
  "title": "电网设备：外部需求激增",
  "body": "美国电网迎来史诗级扩建，750亿美元…",
  "chart": {
    "type": "bar",                     // bar | line
    "xLabel": "年份", "yLabel": "变压器出口金额（亿美元）",
    "categories": ["2019","2020","2021","2022","2023","2024","2025"],
    "series": [{ "name": "变压器出口金额", "values": [48.2,52.7,58.9,65.4,72.1,67.0,90.36] }]
  },
  "source": "资料来源：长城证券《…》作者：…"
}
```
> ⚠️ 图表是最难做成纯数据驱动的部分：柱高/折线点位由 values 计算。A=bar、B=line 说明要支持多种图表类型。这块可能需要一个小的图表渲染层（CSS 高度换算或轻量 SVG 生成）。
> 📝 **图表以自然语言为准，图型不限**（柱/折线/面积/饼/环/散点/堆叠/分组）。上面的 `chart` 结构化对象
> **只是 X/Y 类图（柱/线/面积）的可选快捷写法**，不是通用 schema——饼/环没有轴、散点是两个数值维度，别硬套。
> agent 按图型自选内部数据形状 + 遵守统一配色，数值/类别对不齐或缺数时应问用户、不许编数。详见 SKILL.md「Step 2.5」。

### 5. `buy-fund-cmb` — 买基金 来招行（singleton，**完全固定**）
品牌条。**内部任何东西都不改**——文字、图标、样式全锁死。唯一会变的是它的**相对位置**（上方节点高度变化时它随流下移），这由布局自动处理，不是内容。
```json
{}   // 无可变字段
```

### 6. `compliance-notice` — 合规声明（singleton，**文字可变 / 视觉固定 / 假表格**）
**不是全固定。** 4 张费用表（表格一/二/三/四）的文字**与所推荐的产品绑定**：产品名+代码、产品专属备注、各档费率数值，都随产品变。
- **可变**：产品名+代码（`0:1300/0:1302`）、产品备注（`0:1258` 等）、费率数值（`1.50%/0.00%`、`0.15%/0.05%/0.20%`、`…0.50%/0.10%` 等）。
- **固定**：所有标签（申购费/赎回费/管理费/托管费/销售服务费、费用类型/类别、收费方式/费率、份额/持有期限、N<7天/N≥7天…）、字体字号、风险提示 boilerplate。
- ⚠️ **假表格**：MasterGo 上这不是 `<table>`，是用一根根竖线/矩形（`路径12备份N` 等 PATH）**拼画**出来的网格。网格 PATH 全固定；文字是**绝对定位叠在网格格子上**的。
- ⚠️ **此组件是"自适应高度"的例外**：因为文字必须与固定网格严格对齐，**不能**改成 flow/自适应，否则费率数字会错格。这里保留绝对定位、只原位替换文字值（长度尽量相当）。

费用表与产品耦合，故按产品分组：
```json
{
  "riskWarning": "风险提示：本页面观点仅代表当时观点…（固定 boilerplate）",
  "products": [
    {
      "nameCode": "天弘中证电网设备主题指数C  025833",
      "note": "注：天弘中证电网设备主题指数发起C不收取申购费用。",
      "txnFees": { "申购费": "1.50%\n0.00%", "持有期限": "N<7天\nN≥7天" },
      "assetFees": { "管理费": "0.15%", "托管费": "0.05%", "销售服务费": "0.20%" }
    },
    {
      "nameCode": "华夏中证电网设备主题ETF发起式联接C  025857",
      "note": "注：投资者选择认购/申购C 类基金份额的，不收取认购费、申购费…",
      "txnFees": { "赎回费": "1.50%\n0.00%", "持有期限": "N<7天\nN≥7天" },
      "assetFees": { "管理费": "…0.50%", "托管费": "…0.10%" }
    }
  ]
}
```
> 各产品的费表结构不完全一致（指数 C 有销售服务费百分比，ETF 联接 C 的管理/托管费是描述式文本）。schema 需按产品实际费表灵活填，标签结构不动。

### 7. `related-products` — 相关产品（singleton 容器 + repeatable 产品卡）
标题「相关产品」+ 产品卡数组。产品卡本身也是 **repeatable**（每期几只产品就几张卡）。
```json
{
  "title": "相关产品",
  "products": [
    { "name": "天弘中证电网设备主题指数C", "code": "025833",
      "riskTag": "R5高风险", "returnLabel": "成立以来", "cta": "小试一笔" },
    { "name": "华夏中证电网设备主题ETF发起式联接C", "code": "025857",
      "riskTag": "R5高风险", "returnLabel": "成立以来", "cta": "小试一笔" }
  ]
}
```
> `riskTag` 随产品固有属性；`cta`「小试一笔」是品牌固定用语。

## 小结

- **CSS 种类从 8 → 7**（recommendation A/B 合并成 1）。
- **2 种是可复制的**：`recommendation`（推荐理由）、`related-products` 内的产品卡。
- **蓝渐变白卡骨架**是 hotspot / market / recommendation / related-products 共用的 **base skeleton**。
- **`buy-fund-cmb` 完全固定**，只随流移动位置。
- **`compliance-notice` 文字可变、视觉固定**：随产品换费率/产品名；假表格（PATH 网格）不动。
- **自适应高度的适用范围**：recommendation / hotspot / market / related-products 用 flow、高度随内容长。**compliance 是例外**——假表格必须绝对对齐，保持定位、只换文字。
- **跨模块产品一致性**：同一只产品会在 `recommendation`、`related-products`、`compliance-notice` 三处出现（名称/代码/费率要一致）。换产品时三处联动。
- 最大技术难点：`recommendation.chart` 的数据驱动渲染（bar/line 多类型）。
