## 0. 背景与目标

`scripts/prepare/split_modules.py` 目前输出的每个模块 `node` 树是 **getDsl 原样**：
样式按引用存（`font_0:80` / `paint_0:83` / `effect_0:78`），实际值在 `styles` 子集里，
svg / 导出图 / D2C 分散在 `assets`、`d2c` 里，节点自身字段随类型时有时无。

**目标**：把所有 MCP 源里关于每个节点的信息，合并进一个**自包含节点对象**；
字段集合**固定且最全**，有数据填、没数据填默认值（`null` / `false` / `{}` / `[]`），
方便后续直接从 JSON 生成 CSS。

**核心原则**：固定最大 Schema。字段位永远都在（跨设计稿稳定），
解引用后的值 + getDsl 原始字段**都保留**（如 `textColor` 原始数组与解析后的 `textRuns` 并存）。

## 1. 各 MCP 源能提供什么（本次抓取实测）

| 源 | 贡献 | 能否按 nodeId 干净合并 |
|---|---|---|
| **01-getDsl** | 结构 + `layoutStyle` + `fill/effect/opacity/stroke*(4)/mask/borderRadius/path/text*`；`layoutStyle.rotate`/`rotateX`（36 非零旋转） | ✅ 主干 |
| **dsl.styles 字典** | font 对象 / gradient / box-shadow 字符串的**实际值**（节点里只存 key） | ✅ join 解引用 |
| **05-getDesignTexts** | 长文本还原（占位符→全文） | ✅ 已接（split 阶段） |
| **05-extractSvg(35)** / **05-getDesignSvgs(13)** | 节点级 svg 字符串 | ✅ 按 nodeId |
| **07-getD2c `image`** | 节点导出清单（文件名带 nodeId） | ✅ 按 nodeId |
| **07-getD2c `code`** | 整页 HTML/CSS；唯一提供 `blendMode`（全页仅 4 个）与 `overflow:hidden`（仅页面根 1 个） | ⚠️ 挂叶子元素、无 nodeId，尽力匹配 |
| **02-getMeta** | 本次为空（`<info></info>`） | — 无贡献 |
| **03-getDesignSections** | 每节点字段是 getDsl 子集 | — 字段无新增 |

**关键事实**：
- `rotation` 在 getDsl 原生（`layoutStyle.rotate`），精确、免几何匹配。
- `blendMode` **只在 D2C**，全页仅 4 个，需尽力匹配。
- `clipsContent` 各源基本没有：getDsl 未吐该字段；D2C 里 `overflow:hidden` 仅页面根 1 处，
  模块内节点全部推不出 → 默认 `false`。
- `componentDefinition`：本次 `dsl.components=[]`、`componentDocumentLinks=[]` 均空 → 默认 `{}`。

## 2. 固定节点 Schema（每个节点都有全部 key）

```jsonc
{
  // 身份 / 结构
  "id":"0:1317", "name":"位图一", "type":"LAYER",
  "parentId":"2:796"|null, "childrenIds":[...], "ancestorPath":["0:917","2:796"], "depth":2,
  "sources":["getDsl","getD2c-image", ...],          // 命中哪些源

  // 几何
  "layoutStyle":{"width":90,"height":293,"relativeX":174.5,"relativeY":-119.65,"rotate":90,"rotateX":180}, // getDsl 原样
  "bounds":{"x":..,"y":..,"width":90,"height":293},  // 绝对坐标（累加父 relativeX/Y 算出）

  // 视觉（解引用值 + 原始 ref 并存）
  "fill":"linear-gradient(...)"|"#RRGGBB"|{"imageUrl":".."}|null,  // 解引用；原 ref 见 styleRefs.fill
  "opacity":1|null,
  "stroke":{"color":..,"type":..,"align":..,"width":..}|null,      // 解引用便捷对象
  "strokeColor":null,"strokeType":null,"strokeAlign":null,"strokeWidth":null, // getDsl 原始扁平字段
  "effect":{"raw":[".."],"boxShadow":[".."],"blur":[".."]}|null,   // 解引用
  "borderRadius":null, "mask":null, "path":null,
  "clipsContent":false,          // D2C overflow:hidden 推断；无则 false

  // 旋转（getDsl 原生）
  "rotation":90,                 // = layoutStyle.rotate（真值）
  "rotationX":180,               // = layoutStyle.rotateX
  "rotationY":null,              // 本次无，留位
  "blendMode":"screen"|null,     // 仅 D2C，尽力匹配，对不上 null
  "d2cMatch":"img-src"|"geometry"|null,  // blendMode 怎么对上的

  // 文本（原始 + 解析并存）
  "text":"..."|null,
  "textColor":[{"start":0,"end":9,"color":"paint_.."}]|[],          // getDsl 原始数组
  "textRuns":[{"text":..,"start":..,"end":..,"font":{..}|null,"color":".."|null}]|[], // 解析后
  "textAlign":null, "textMode":null,

  // 资源
  "hasSvg":true|false,           // svg 串集中放模块 assets.svgs[nodeId]
  "svg":null,                    // 保留字段位，串在 assets.svgs
  "exportImage":{"fileName":"位图一-0-1317.svg","format":"SVG",...}|null, // d2c image 清单

  // 组件 / 溯源 / 质量
  "componentDefinition":{},      // 本次 components 为空，留位
  "styleRefs":{"fill":"paint_1:4241","effect":"effect_..","font":"font_..","stroke":"paint_..","textColors":["paint_.."]}, // {} 若无
  "captureStatus":"complete"|"partial",  // partial = 该有的 D2C blend 没对上

  "children":[ /* 递归的解析后节点 */ ]
}
```

模块顶层新增：`assets.svgs = {"0:925":"<svg ...>...</svg>", ...}`（节点 `hasSvg=true` 时按 id 查）。

## 3. 逐字段解析规则（来源 + 回退默认）

| 字段 | 来源 / 做法 |
|---|---|
| parentId / childrenIds / ancestorPath / depth | 遍历 getDsl 树推导 |
| bounds | 累加各级父 `relativeX/Y` + 自身 w/h，算绝对坐标 |
| fill / stroke.color | `paint_` ref → `styles[ref].value`；单值取字符串，含 `url` → `{imageUrl}` |
| strokeColor/strokeType/strokeAlign/strokeWidth | getDsl 原始扁平字段，原样保留 |
| effect | `effect_` ref → value 是 CSS 字符串数组；解析出 boxShadow/blur，保留 raw |
| textRuns[].font | `font_` ref → `styles[ref].value`（字体对象） |
| textRuns[].color | textColor[].`paint_` ref → 解析 |
| textColor | getDsl 原始数组，原样保留 |
| text | textRuns 文本拼接（占位符 split 阶段已还原） |
| rotation / rotationX / rotationY | `layoutStyle.rotate`（无则 0）/ `rotateX` / null |
| clipsContent | D2C 匹配 div 的 `overflow:hidden` 推断；无则 false |
| blendMode / d2cMatch | 见第 4 节 |
| hasSvg / svg / assets.svgs | 模块 `assets.nodeSvgs`/`sectionSvgs` 里按 id 取，串下沉到 `assets.svgs[id]`，节点标 `hasSvg` |
| exportImage | 模块 `d2c.exportImages` 里文件名含该 nodeId 的条目 |
| componentDefinition | 本次空 → `{}` |
| styleRefs | 保留节点用到的所有原始 ref（fill/effect/font/stroke/textColors） |
| sources | 该 nodeId 在哪些源里出现过 |
| captureStatus | complete；该有的 D2C blend 没对上 → partial |

## 4. blendMode（唯一需要 D2C，本页仅 4 个）

在模块 `d2c.html` 上锚定，只为补 `blendMode`（顺带 `clipsContent` 的 overflow 推断）：

1. **img-src 精确**：`<img src="asset/images/名字-0-921.svg">` 文件名带 nodeId →
   把该 img（及定位祖先）的 `mix-blend-mode` 挂到对应节点，`d2cMatch="img-src"`。
2. **几何匹配**：把 d2c.html 解析成 DOM，累加 left/top（px 与 %，% 相对祖先内容框）算绝对 bounds；
   带 `mix-blend-mode` 的元素找 `bounds` 最接近的节点（`|dx|+|dy|+|dw|+|dh| < 阈值`），`d2cMatch="geometry"`。
3. 对不上 → 节点 `blendMode=null` / `captureStatus="partial"`，元素记进 `meta.d2cUnmatched`（值 + bounds）。

> rotation 已从 getDsl 原生拿到（精确），**不需要**几何匹配。D2C 只补 blendMode 这 4 个。

## 7. 已知边界（诚实说明）

- `clipsContent`：本次数据除页面根外无来源，模块内节点全 `false`。要真值需 fetch 阶段让 MCP 吐该字段。
- `blendMode`：全页仅 4 个，img-src 能精确对上一部分，其余靠几何匹配，可能个别对不上——对不上会显式列入 `meta.d2cUnmatched`，不静默丢。
- `componentDefinition`：本设计稿无组件实例，恒为 `{}`。
- `rotationY`：getDsl 无此维度，恒为 `null`。
