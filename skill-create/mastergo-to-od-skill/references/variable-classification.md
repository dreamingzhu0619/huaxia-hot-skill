# Fixed / Variable 分类判定规则

> 适用：Step 4 中对每个模块的每个节点判定哪些内容会变、哪些不变。
> 这是整个 pipeline 最核心的判断环节。脚本做自动标记，AI 做语义确认，最终以用户意见为准。

---

## 三档分类

| 标记 | 含义 | 对产物的影响 |
|------|------|------------|
| `fixed` | 完全不变 | 样式进 styles.css；装饰进 decorations.html；固定 TEXT 原文保留在 template.html |
| `variable` | 内容可变（文本替换） | template.html 中暴露为 `{{slot}}` |
| `variable-all` | 整块可变（结构+内容都可变） | 整体暴露为 `{{chart}}` 或 `{{slot}}`，内部不做逐节点 diff |

---

## 核心判断问题

> **下一期再做一张同样的 H5，这个内容会变吗？**

这个问题不能只看文本内容本身，要追问内容从哪里来、在页面里担任什么角色。

---

## 判断原则

### 原则一：按内容来源判断

内容的来源决定了它会不会变。

| 来源 | 判定 | 原因 |
|------|------|------|
| **实时行情数据**（收盘价、涨跌幅、指数点位） | `variable` | 每期必然不同 |
| **新闻资讯正文** | `variable` | 每期重写 |
| **产品名称**（"华夏中证红利低波动ETF发起式联接C"） | `variable` | 每期推荐的产品可能不同 |
| **产品代码**（"021483"） | `variable` | 随产品变化 |
| **日期/时间戳**（"2026.03.05"、"截止3月5日"） | `variable` | 每期更新 |
| **数据来源标注**（"数据来源：Wind，截至2026.5.22"） | `variable` | 来源和日期都变 |
| **费率具体数值**（"0.15%"、"0.20%"、"0.05%"） | `variable` | 换产品后费率可能不同 |
| **研究报告引用**（"资料来源：华泰研究"） | 看情况 | 如果每期引用不同报告 → `variable`；如果是固定免责 → `fixed` |
| **品牌标语**（"买基金 来招行"） | `fixed` | 品牌资产，不随营销周期变化 |
| **风险提示全文**（"基金有风险，投资须谨慎…"） | `fixed` | 合规条文，逐字审核通过，不随意改 |
| **费用表格列头**（"管理费"、"托管费"、"销售服务费"） | `fixed` | 费用类别名称是固定的 |
| **费用表格标签**（"费用类别"、"收费方式/年费率"） | `fixed` | 表格结构标签 |

### 原则二：按文本功能判断

不看文本内容，看它在页面里担任什么角色。

| 功能 | 判定 | 原因 |
|------|------|------|
| **模块标题**（概括本期主题的标题类文字） | `variable` | 主题每期不同 |
| **副标题/小标题** | `variable` | 随主题变化 |
| **正文段落**（展开论述的具体内容） | `variable` | 内容每期不同 |
| **装饰文字**（角标、标签、功能按钮文字） | `fixed` | 模块自身固有的 UI 元素，不描述具体产品/热点 |
| **CTA 按钮文案**（"小试一笔"、"立即查看"） | `fixed` | 品牌固定用语，不是产品数据 |
| **合规条文** | `fixed` | 法律文本，不随营销周期变化 |
| **表格列头/行标签** | `fixed` | 结构信息 |
| **表格数据值** | `variable` | 具体数值会变 |

### 原则三：按视觉元素类型判断

| 类型 | 判定 | 原因 |
|------|------|------|
| **PATH**（贝塞尔曲线、图标、装饰线） | `fixed` | 设计稿的视觉语言，不随内容变 |
| **LAYER**（带填充/描边的矩形/圆形） | `fixed` | 装饰元素，除非是图表中的数据柱 |
| **SVG_ELLIPSE** | `fixed` | 装饰元素 |
| **FRAME**（容器/编组） | `fixed` | 结构框架，除非内含图表数据 |
| **BITMAP / 位图** | `fixed` | 装饰图片或 logo |
| **TEXT** | 逐一判断 | 唯一需要逐节点分析的节点类型 |

#### 特殊情况：纯 PATH 组成的 GROUP

如果一个 GROUP 的所有子节点（含递归）全部是 PATH 类型 → **极大概率是 logo 或矢量装饰**。整组标 `fixed`，完整进入 decorations.html。

```
示例：
银行结束语 → GROUP → [PATH×49, LAYER×2]
判定：49 个 PATH 子节点，0 个 TEXT → 整个模块 fixed → 完整进入 decorations.html
```

#### 特殊情况：固定装饰 GROUP（含 TEXT 但整体 fixed）

如果一个 GROUP 内含 TEXT，但 TEXT 的内容是模块固有的装饰文字（不是产品数据、不是热点内容），则**整个 GROUP → fixed → 整体进入 decorations.html**。

```
示例：
0:35 GROUP "热点速递拷贝"
├── PATH / LAYER（圆角矩形容器、形状、位图）
├── 0:46 TEXT "热点速递"  ← 这是模块自带的角标标签，不是正文内容
└── 0:47 PATH Gradient Overlay

判定：整个 0:35 GROUP → fixed → 完整进入 decorations.html
```

```
示例：
0:53 GROUP "按钮"
├── 0:54 LAYER（红色圆角背景）
└── 0:55 TEXT "小试一笔"  ← 品牌固定 CTA 文案，不是用户填的数据

判定：整个 0:53 GROUP → fixed → 完整进入 decorations.html
```

**识别方法**：读 TEXT 节点的内容 + 节点名 + 所在位置，问自己：**"这个文字是描述具体产品/热点/数据的，还是模块本身自带的 UI 标签？"**

| 问题 | 判定 |
|------|------|
| 这个文字是产品名/代码/收益率吗？ | 如果是 → variable |
| 这个文字是描述本期热点/市场行情吗？ | 如果是 → variable |
| 这个文字是数据来源/日期吗？ | 如果是 → variable |
| 这个文字是角标/按钮/标签/功能名吗？ | 如果是 → fixed |
| 这个文字换成另一个产品，还需要改吗？ | 不需要 → fixed |

### 原则四：看同一模块内 TEXT 的"邻居关系"

把同一模块里的全部 TEXT 节点拉出来对比看：

1. **标题 + 正文**：标题和正文通常同为 `variable`
2. **标签 + 值**：标签（"变压器出口金额"）通常是 `fixed`，值（"90.36"）是 `variable`
3. **表格**：表头（列名）是 `fixed`，数据单元格是 `variable`
4. **图表坐标轴**：轴标签和年份刻度是 `fixed`（坐标系不变），数据柱是 `variable-all`

## 脚本自动判定规则

以下内容脚本自动标 `fixed`，不需要 AI 判断：

| 规则 | 示例 | 逻辑 |
|------|------|------|
| `type` ≠ `TEXT` | PATH / LAYER / BITMAP / SVG_ELLIPSE / FRAME | 视觉元素永不变 |
| 含「风险」「投资须谨慎」「基金合同」「招募说明书」「不保证」 | 风险提示全文 | 合规关键词 |
| 含「管理费」「托管费」「赎回费」「申购费」「销售服务费」 | 费用表头 | 费用类别名称 |
| 含「收费方式」「年费率」「费用类别」 | 表格列头 | 表格结构固定 |

以下内容脚本自动标 `variable`：

| 规则 | 示例 | 逻辑 |
|------|------|------|
| 文本内容匹配日期格式 `\d{4}\.\d{1,2}\.\d{1,2}` | "2026.5.22" | 日期每期更新 |
| 文本内容为纯数字 + `%` | "2.80%" / "-16.92%" | 收益率数据 |
| 文本内容为纯数字（6位） | "021483" | 产品代码 |
| 文本内容含「数据来源」「资料来源」「截至」 | "数据来源：Wind，截至2026.5.22" | 来源标注含日期 |

## AI 判定流程

脚本自动标记完成后，剩下的 TEXT 节点进入 AI 判定。

**判定顺序**：

1. 逐模块读 `singleton_texts.json` 中 `needsReview` 的 TEXT 清单
2. 对每个 TEXT，回答三个问题：
   - **这个文字从哪里来？**（行情数据？用户填写？品牌方提供？合规部门？）
   - **这个文字在页面里担任什么角色？**（标题？正文？标签？按钮？角标？）
   - **下一期换内容，这个文字需要改吗？**
3. 呈现判定清单给用户确认

**呈现格式**：

| 节点ID | 模块 | 当前文本 | 建议分类 | 判断依据 |
|--------|------|----------|----------|----------|
| 0:57 | 产品卡 | 华夏中证红利低波动ETF发起式联接C | `variable` | 产品名称，换产品就变 |
| 0:55 | 产品卡 | 小试一笔 | `fixed` | 品牌 CTA 文案，模块固有元素 |
| 0:46 | 热点速递 | 热点速递 | `fixed` | 角标装饰文字，模块固有标签 |
| 0:33 | 热点速递 | 随着AI科技板块加速上涨… | `variable` | 新闻正文，每期重写 |
| 0:76 | 推荐理由A | 左手红利 右手低波 | `variable` | 推荐标题，每期不同 |
| 0:152 | 推荐理由A | 压舱石 | `variable` | 产品标签，描述产品特征 |
| 0:177 | 推荐理由A | 资料来源：华泰研究。 | `variable` | 来源标注，每期可能换 |
| 0:401 | 风险警告 | 风险提示：基金非存款… | `fixed` | 合规条文，逐字审核 |
| 0:403 | 风险警告 | 0.15% | `variable` | 管理费具体数值，换产品可能变 |
| 0:407 | 风险警告 | 管理费 | `fixed` | 费用类别名称（脚本自动标） |

## 用户确认

用户逐模块确认或调整。**用户意见与启发式规则冲突时以用户为准。**

确认后，判定结果回写到模块 JSON 或单独的 variability 配置文件中，Step 5 的脚本读取这些标记来生成产物。
