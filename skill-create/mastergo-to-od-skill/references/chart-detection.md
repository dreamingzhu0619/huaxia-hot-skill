# 图表区域识别规则

> 适用：Step 4d 中识别模块内的图表区域，将其标记为 `variable-all`，整体暴露为 `{{chart}}` 插槽。
> 原则：图表是数据驱动的视觉元素，内容、形状、数值全部会变，不应逐节点 diff。

---

## 为什么需要专门识别图表

图表在 MasterGo 设计稿中的表现与普通装饰元素不同：

- **普通装饰**：固定的 PATH/LAYER，不随内容变化 → 机械提取到 decorations.html
- **图表**：由大量 LAYER/PATH 组成（柱状图的柱子、折线图的点位、饼图的扇区），但这些形状的尺寸/位置/颜色随数据变化 → 必须识别为 `variable-all`，整体暴露为 `{{chart}}`

如果把图表误判为固定装饰，换一期数据后图表不会更新，渲染结果就是错误的。

---

## 识别信号（由强到弱）

### 强信号：附近有"数据来源"类文本

**这是最强的判断依据。** 如果一个 GROUP 附近（同级或兄弟节点中）存在含以下关键词的 TEXT，该 GROUP 极有可能是图表：

| 关键词 | 示例 |
|--------|------|
| "数据来源" | "数据来源：Wind，截至2026.5.22" |
| "资料来源" | "资料来源：华泰研究。" |
| "参考来源" | "参考来源：东方财富资讯" |
| "统计区间" | "统计区间：2021.5.23-2026.5.22" |
| "数据截止" | "数据截止2026年3月31日" |
| "注：" | "注：均使用全收益指数（含分红）" |

**检测方式**：遍历模块内所有 TEXT 节点，检查 `text` 字段是否匹配上述关键词。如果命中，回溯到该 TEXT 的前一个兄弟 GROUP → **初步标记为图表候选。**

### 中信号：GROUP 内含大量形状元素

如果一个 GROUP 满足以下条件，很可能是图表：

| 条件 | 含义 |
|------|------|
| GROUP 内含 ≥ 5 个 LAYER（矩形） | 柱状图的柱子、表格的单元格 |
| GROUP 内含 ≥ 3 个 PATH | 折线图的线段、饼图的分割线 |
| GROUP 内含 SVG_ELLIPSE | 饼图/环形图的扇区 |
| GROUP 内含 ≥ 8 个 TEXT（数值类） | Y 轴刻度、数据标签 |

**检测方式**：递归统计 GROUP 子树内各类型节点的数量。

### 辅助信号

| 信号 | 说明 |
|------|------|
| 同级或附近有标题类 TEXT | "现金分红统计"、"一季度保险公司持股汇总（按行业）" |
| `name` 含"图表""统计""汇总""分布""对比""走势"等词 | 设计师自定义的名称 |
| GROUP 内部结构在不同实例间不同（合并组内对比） | 有的推荐理由有柱状图、有的有饼图、有的有表格 → 结构不同说明是可变内容 |

---

## 识别流程

```
遍历模块内所有 GROUP 节点：
  │
  ├── 1. 检查附近 TEXT 是否含"数据来源"/"资料来源"/"统计区间"等
  │      │
  │      └── 是 → 图表候选 ✓（强信号，直接命中）
  │
  ├── 2. 统计 GROUP 子树内节点类型分布
  │      │
  │      ├── LAYER ≥ 5 且 TEXT ≥ 5 → 柱状图/表格候选
  │      ├── PATH ≥ 3 且 SVG_ELLIPSE ≥ 2 → 饼图候选
  │      └── 不满足 → 不是图表
  │
  ├── 3. 检查组内 TEXT 的文本内容
  │      │
  │      ├── 大量数值类文本（如 "7200"、"7.5%"、"2016"） → 图表候选
  │      └── 描述性长文本 → 不是图表（是正文）
  │
  └── 4. AI 综合确认
```

## 图表判定后的处理

一旦判定为图表 GROUP：

| 处理方式 | 说明 |
|----------|------|
| **标记** | 整个 GROUP → `variable-all` |
| **template.html** | `<div class="...">{{chartDescription}}</div>` 或 `{{chart}}` |
| **不进入 decorations.html** | 图表内部元素（柱子、线条、数值）不提取为固定装饰 |
| **不逐节点 diff** | 不对比柱子数量、高度差等内部细节 |

### 图表数据输入方式

图表数据以自然语言描述为主，用户口述即可：

```
示例：
"画一个柱状图，X轴是年份2016-2026，Y轴是现金分红总额，数值从1200亿到7200亿递增，同时叠加折线表示股息支付率从1.5%到7.5%"
```

`content.template.json` 中对应字段：

```json
{
  "chart": "柱状图：2016-2026年现金分红总额（1200→7200亿），叠加折线：股息支付率（1.5%→7.5%）"
}
```

## 图表中固定元素的处理

图表内也有**部分元素是固定的**，不应全部丢弃：

| 固定元素 | 处理 | 说明 |
|----------|------|------|
| 坐标轴标签（2016, 2018, 2020...） | 进 template.html 作为固定 DOM 结构 | 年份刻度是固定的 |
| 列头/行标签（"指数名称"、"近5年年化收益率"） | 进 template.html 作为固定 DOM 结构 | 表格结构标签 |
| 图例（"红利低波全收益"、"300收益"） | 进 template.html 作为固定 DOM 结构 | 对比标的名称 |
| 数据柱/线条/扇区 | `variable-all`，由 OD agent 重新渲染 | 数据变 → 形状变 |

**策略**：图表区域采用**描述驱动**而非**机械还原**的方式。不从 PATH/LAYER 提取具体的 SVG 形状，而是提取"这个图表在展示什么数据关系"——由 OD agent 根据描述重新渲染。

## 示例：huaxia-hot-citc

### 推荐理由 B 中的柱状图+折线图

```
0:202 GROUP 组5815（图表区域）
├── 0:204~0:217 LAYER×14（柱状图背景网格 + 柱子）
├── 0:218 PATH 形状6（折线）
├── 0:220~0:234 TEXT×15（数值标签："7200""6000"..."2016""2026"）
└── 附近 TEXT：
    ├── 0:198 "现金分红总额（亿）"  ← 图表标题
    ├── 0:201 "股息支付率"           ← 图表标题
    └── 0:192 "数据来源：Wind，截至2026.5.22"  ← 强信号 ✓

判定：满足强信号（数据来源）+ 中信号（14 LAYER + 1 PATH + 15 TEXT） → 图表 → variable-all
```

### 推荐理由 C 中的表格+饼图

```
0:287 GROUP 组5809（表格区域）
├── 0:288~0:292 LAYER×5（表格背景）
├── 0:293~0:309 TEXT（表头 + 数据值）
│   ├── "指数名称"、"近5年年化收益率"、"近5年年化波动率"、"近5年最大回撤"  ← 列头
│   ├── "红利低波全收益""300收益""500收益""科创50（全）"  ← 行标签
│   └── "10.14%""1.14%"...  ← 数据值
└── 附近 TEXT：
    ├── 0:311 "数据来源：Wind，统计区间：2021.5.23-2026.5.22"  ← 强信号 ✓
    └── 0:310 "注：均使用全收益指数（含分红）"  ← 强信号 ✓

判定：图表 → variable-all

固定部分（进入 template.html）：
  - 表格列头（"指数名称"、"近5年年化收益率"、"近5年年化波动率"、"近5年最大回撤"）
  - 行标签（"红利低波全收益"、"300收益"、"500收益"、"科创50（全）"）

可变部分（{{chart}} 插槽）：
  - 具体数值（10.14%、1.14%、7.24%、6.80%...）
  - 饼图的扇区比例

0:256 GROUP 组5808拷贝2（饼图区域）
├── 0:257 LAYER 矩形1001（卡片背景）
├── 0:261 SVG_ELLIPSE 椭圆560（圆环）
├── 0:262 PATH 椭圆560拷贝2（圆环弧线）
├── 0:263 TEXT "占比超60%"  ← 数值
├── 0:260 TEXT "红利资产"  ← 标签
└── 附近 TEXT：
    └── 0:255 "数据来源：Wind，截至2026.3.31"  ← 强信号 ✓

判定：饼图 → variable-all
```

## AI 确认清单

对脚本标记的每个图表候选，AI 需确认：

- [ ] 附近是否有"数据来源"/"资料来源"/"统计区间"/"注："等关键词？
- [ ] GROUP 内部是否包含 ≥ 5 个 LAYER 或 ≥ 3 个 PATH/SVG_ELLIPSE？
- [ ] GROUP 内的 TEXT 是否以数值类内容为主（而非描述性段落）？
- [ ] 如果以上满足 → 整个 GROUP 是图表 → `variable-all` → template.html 中暴露为 `{{chart}}`

**特别注意**：表格中的固定标签（列头、行标签、坐标轴）需要保留在 template.html 中作为固定 DOM 结构，只有数据值进入 `{{chart}}` 描述。
