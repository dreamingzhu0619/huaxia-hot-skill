# 华夏行业 H5 Skill — 工作流设计

## 背景

当前 `mg-to-od-skill-v2` 试图做通用的 MasterGo 设计稿 → H5 Skill 提取管道，但因为设计稿结构变化太复杂，通用方案难度极大。因此退一步，先针对 `huaxia-hot-industry`（半导体行业 H5）这一个具体设计稿做提取。

**最终目标**：提取这个 H5 的固定制式，然后灵活更换内容。比如这次是半导体，下次输入光伏行业的数据，就能生成相应的 H5。

## 当前数据状态

MasterGo 设计稿已通过 MCP 抓取并拆分为 18 个模块（按页面根节点的直接子节点拆分），存储在 `data/modules/` 中。

**问题**：自动拆分（按 DOM 树直接子节点）并不符合业务语义。比如"关联产品"板块实际分散在模块 08、09、10、11 等多个模块中。

---

## 总体工作流（2 步）

### Step 1: 合并 Frame

把 18 个模块合并为 6 个业务 Frame。合并映射由我直接指定（因为这个 skill 只针对当前设计稿）。

**输出**：`data-output/frames-merged/` — 每个业务 Frame 一份合并后的完整数据

### Step 2: 判断类型并处理

对每个 Frame 判断类型（固定型/模板型/内容型），走对应的处理逻辑：

- **固定型** → 完整保留 HTML/CSS，不做任何标记
- **模板型** → 提取组件模板结构 + 检测重复实例 + 标记可变文字 slot + 提取文字样式
- **内容型** → 提取固定框架（标题栏、背景框）+ 识别内容区 + 提取文字样式 + 内容区用占位符标记

**输出**：`data-output/frames/` — 每个 Frame 的最终产物（template.html + styles.css + schema.json + assets/）

中间产物和最终产物都放在 `data-output/` 下。

## 三种 Frame 类型定义

**共同点**：模板型和内容型都需要提取文字样式（字体、字号、颜色、行高等），这不是区分两者的依据。

**核心区别**：

| | 模板型 | 内容型 |
|-------|---------|--------|
| 架构 & 组件 | **完全固定**，组件布局不可变 | **只有框架固定**（标题栏、背景框），内部内容排布自由 |
| 可变内容 | **只换文字**，不增删组件 | **图文排布自由变化**——原稿一张图+一段话，新版可以两张图+两段话 |
| 举例 | 产品卡：产品名/代码/风险等级的位置和样式永远不变 | 产业逻辑拆解：内容区里可以随意画图、填文字、调整排版 |

### 固定型

除了相对位置，其他都不变。

- **处理方式**：完整保留 HTML/CSS/资源，不做 slot 标记
- **输出**：完整的 HTML 片段 + CSS + 资源文件
- **实例**：风险提示

### 模板型

架构和组件完全固定，只需要替换其中的可变文字。

- **处理方式**：
  1. 检测并提取重复实例的模板结构
  2. 用 `{{slot_name}}` 标记可变文字位
  3. 提取所有文字样式（标题样式、正文样式、标签样式等）
  4. 生成 schema.json 定义每个 slot 的类型和约束
- **输出**：模板 HTML（含 slot 标记） + CSS + schema.json + 资源文件
- **实例**：关联产品、头图 Banner

### 内容型

只有框架（标题栏、背景框、装饰）是固定的，内容区里的图文排布完全自由。

- **处理方式**：
  1. 保留固定框架（标题栏、装饰、背景框）的 HTML/CSS
  2. 提取框架内使用的文字样式（供内容区参考使用）
  3. 内容区用占位符标记，用户可自由填入任意 HTML
  4. 记录内容区的尺寸约束（宽度、最小高度、padding 等）
- **输出**：框架 HTML + CSS + 文字样式清单 + schema.json（定义内容区）+ 资源文件
- **实例**：近期热点、产业投资价值、产业逻辑拆解

### 文字可变性判断原则

**核心规则**：只要文字里包含行业名称（如"半导体"），那它就是可变的。因为换一个行业（如"光伏"），所有涉及行业名的文字都会跟着变。

**固定文字**（不随行业改变）：
- "行业直击"
- "捕捉热点事件，拆解产业逻辑，精选关联产品"
- "近期热点"
- "关联产品"
- "产业基石" / "价值核心" / "需求源泉"（产业链标签）
- "查看更多"
- "风险提示"

**可变文字**（随行业改变或每次不同）：
- 行业名称（"半导体" → "光伏"）
- 所有包含行业名称的标题（"半导体未来投资价值如何？"、"半导体是什么？"）
- 热点新闻的具体内容（标题、正文、来源）
- 产业链各环节的具体标题（"上游-设备与材料" → "上游-硅料与硅片"）
- 产品名称、代码等具体数据

## 详细工作流

### Step 1: 合并 Frame

**输入**：
- `data/modules/_index.json` + `data/modules/*.json`（模块数据）
- `data/input/frame-merge-map.json`（用户填写的合并映射）

**处理**：脚本读取 `frame-merge-map.json`，按映射将指定模块的 `node` 树、`d2c` HTML、`assets` 合并到对应业务 Frame。

**输出**：`data-output/frames-merged/` — 每个 Frame 一份 JSON

### Step 2: 判断类型并处理

**输入**：
- `data-output/frames-merged/*.json`（Step 1 产物）
- `data/input/frame-types.json`（用户填写的类型定义 + 子结构）

脚本读取 `frame-types.json`，对每个 Frame 按指定类型走对应处理逻辑。

---

#### 固定型处理（06-risk-warning）

完整保留 HTML/CSS/资源，不做任何 slot 标记。

**输出**：
- `template.html` — 完整的 D2C HTML 片段
- `styles.css` — 该 Frame 的样式
- `assets/` — 图片、SVG 等资源
- `schema.json` — 空（无需填充）

---

#### 模板型处理（01-banner、03-products）

**处理步骤**：

1. **提取组件结构**：从 D2C HTML 中提取 Frame 的完整 DOM 结构
2. **检测重复实例**：在 normalized tree 中对兄弟节点做结构 diff——节点类型相同、子节点结构相同、仅 TEXT 不同的 → 同一模板的重复实例
3. **标记可变文字 slot**：将可变文字替换为 `{{slot_name}}`，重复实例用循环结构包裹
4. **提取文字样式**：从可变文字节点提取 font-family、font-size、color、line-height、letter-spacing 等，写入 schema.json

**输出**：
- `template.html` — 含 `{{slot}}` 标记的模板 HTML
- `styles.css` — 组件样式
- `schema.json` — 定义了每个 slot 的名称、类型、文字样式、约束

```json
// 以 03-products 为例
{
  "type": "template",
  "repeatableInstances": [
    {
      "key": "items",
      "templateName": "product-card",
      "slots": [
        { "name": "name", "type": "text", "style": "product-name" },
        { "name": "code", "type": "text", "style": "product-code" },
        { "name": "risk", "type": "text", "style": "risk-badge" }
      ]
    }
  ],
  "textStyles": {
    "product-name": { "fontFamily": "...", "fontSize": 14, "color": "#333", ... },
    "product-code": { ... },
    "risk-badge": { ... }
  }
}
```

- `assets/` — 图片、SVG、图标等固定资源

---

#### 内容型处理（02-hotspot、04-investment-value、05-industry-logic）

**处理步骤**：

1. **提取固定框架**：识别并保留标题栏、背景框、装饰元素的 HTML/CSS
2. **识别内容区**：框架内部的可变区域 → 标记为 content slot（`{{content}}`）
3. **提取文字样式**：从框架中的文字（标题、标签等）提取样式定义，供内容区参考使用

对于 **05-industry-logic**，需要进一步拆分子板块：

| 子板块 | 固定部分 | 内容区 |
|--------|---------|--------|
| 05a-definition | 标题栏样式（"XX是什么？"） | 定义内容 + 图表（自由 HTML） |
| 05b-supply-chain | 标签（产业基石/价值核心/需求源泉） + 标题栏样式 | 产业链各环节内容（自由 HTML） |

上中下游三个子板块共用一个模板（标签文字固定，标题栏样式固定，标题文字可变，内容区自由）。

**输出**（每个 Frame/子板块）：
- `template.html` — 框架 HTML + 内容区占位符 `{{content}}`
- `styles.css` — 框架 + 文字样式
- `schema.json` — 内容区定义 + 文字样式清单

```json
// 以 05b-supply-chain 为例
{
  "type": "content",
  "fixedTexts": {
    "tagLabel": "产业基石"
  },
  "variableSlots": [
    { "name": "sectionTitle", "type": "text", "style": "section-title" }
  ],
  "contentZones": [
    { "name": "content", "type": "html", "description": "自由内容区，可填入任意 HTML" }
  ],
  "textStyles": {
    "section-title": { "fontFamily": "...", "fontSize": 16, ... },
    "tag-label": { ... }
  }
}
```

- `assets/` — 固定装饰元素的图片/SVG

---

### Skill 执行：H5 组装

当用户提供了 `input/content.json` 后，组装最终 H5：

1. 读取 `data-output/frames/frames-index.json` 确定 Frame 顺序
2. 对每个 Frame / 子板块：
   - 加载 `template.html`
   - 读取 `input/content.json` 中对应内容
   - 模板型：将文字值填入 `{{slot}}`
   - 内容型：将用户 HTML 注入 `{{content}}`
3. 按流式布局从上到下拼接
4. 合并 CSS，收集资源

**高度自适应**：Frame 间用流式布局（相对定位 + margin/padding），不使用绝对定位，页面总高度 = 各 Frame 实际高度之和。

### Q1: 重复实例检测策略

- **方案 A**：自动检测（结构 diff 兄弟节点）
- **方案 B**：配置指定（手动标注）

### Q2: 产业链子板块的数量是否可变

当前半导体设计稿的上中下游是 3 个环节。如果换成别的行业（比如光伏的"硅料→硅片→电池→组件→电站"可能有 5 个环节），产业链环节数量是否可变？还是说固定为 3 个？这决定了 supplyChain 是数组（可增减）还是固定 3 个字段。

### Q3: 高度自适应

模板型和内容型 frame 的高度取决于内容的多少，因此不能用绝对定位，需要用流式布局。整个页面是各 frame 从上到下自然堆叠：
- 需要从设计稿中提取每个 frame 的 margin/padding
- D2C 给出的 absolute positioning 需要转换为相对定位
- 页面总高度不固定

### Q4: 模板引擎选择

Phase 5 的 H5 组装需要一个模板引擎。选项：
- **Jinja2**（Python 生态，成熟稳定，支持循环/条件）
- **Mustache**（跨语言，逻辑简单）
- **直接字符串替换**（最简单，但循环逻辑需要自己处理）

建议用 Jinja2。
